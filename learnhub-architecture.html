LearnHub Architecture

Spaced-repetition flashcard platform with AI-powered grading — Next.js frontend + FastAPI backend

Frontend Layer

  • • Next.js 14 with App Router
  • • Server Components + Client Components
  • • shadcn/ui + Radix UI + Tailwind CSS
  • • API Proxy Routes → FastAPI backend
  • • fetchWithRetry (5 retries, exponential backoff)
  • • Client-side: session state, gamification, study flow
  • • NextAuth.js with Credentials Provider

Backend Layer

  • • FastAPI (Python 3.12) on port 3001
  • • Modular routers: auth, user, flashcards, vocabulary
  • • Spaced repetition: SM-2 + TGCL (PyTorch)
  • • AI Grader: COMET wmt22-comet-da + MiniLM semantic gate
  • • Fallback: Levenshtein + diacritics normalization
  • • Pydantic v2 schemas, CORS middleware
  • • HuggingFace Hub model caching (HF_HOME)

Data & AI Models

  • • SQLite: vocabulary, user, review_log, user_badge tables
  • • SQLAlchemy ORM with migrations
  • • COMET (Unbabel/wmt22-comet-da) ~400MB
  • • MiniLM (paraphrase-multilingual-MiniLM-L12-v2) ~90MB
  • • TGCL model (tcgl_learned.pth) with online learning
  • • Combined scoring: COMET × embedding similarity gate
  • • Grading: exact → diacritics → semantic → levenshtein

Key Data Flows

  • • Study: Session → Check Answer → COMET+Embed Score → Rating
  • • Review: Submit → SM-2/TGCL → Update intervals → XP/Badges
  • • Auth: Login → NextAuth JWT → Session → API proxy
  • • Vocabulary: CRUD + Enrich (admin only)
  • • Retry: 5 attempts, 1s base, 1.5x for connection errors
  • • Gamification: XP points, levels, streaks, unlockable badges
  • • ML: Online learning after each review + batch training
